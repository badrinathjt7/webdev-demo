WHAT IS DESIGN?
5 Core Principles Every Developer Needs
Design has rules ‚Äî and they're learnable. You don't need to be an artist.

‚ë† Visual Hierarchy

Size, weight, and colour signal importance. H1 > H2 > body text. The eye should naturally flow top ‚Üí bottom. Big bold text = "read this first."

‚ë° Contrast & Readability

Text must be readable. Light grey on white = invisible. WCAG requires 4.5:1 contrast ratio for body text. Dark text on light background is always safest.

‚ë¢ Whitespace

Empty space is not wasted space. Padding and margins reduce cognitive load and make content scannable. More whitespace = more premium feel.

‚ë£ Consistency

Same button style everywhere. Same spacing scale. Inconsistency makes an app feel broken even if the code is correct. Pick a system and stick to it.

‚ë§ Colour Theory (60/30/10)

60% dominant background ¬∑ 30% neutral surfaces ¬∑ 10% accent colour. Accent is for CTAs only ‚Äî overusing it dilutes its power.

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>Count: <span>{count}</span></h2>
      <button onClick={() => setCount(count + 1)}>
        + Increment
      </button>
      <button onClick={() => setCount(count - 1)}>‚àí Decrement</button>

// ‚îÄ‚îÄ Component 2: TodoList ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
function TodoList() {
  const [todos, setTodos] = useState([]);
  const [input, setInput] = useState('');

  const addTodo = () => {
    if (!input.trim()) return;
    setTodos([...todos, { text: input, done: false }]);
    setInput('');
  };

  return (
    <div>
      <input value={input} onChange={e => setInput(e.target.value)} />
      <button onClick={addTodo}>Add</button>
      {todos.map((t, i) => (
        <p key={i}>{t.text}</p>
      ))}
    </div>
  );
}
